GS.HTTP Privacy Policy

The GS.HTTP mobile app's privacy policy, in accordance with the EU General Data Protection Regulation (GDRP) and the California Consumer Privacy Act (CCPA), includes a 7-point plan:
1.General Provisions & Compliance
2.MIT License & open-sourse audit
3.Data categories & local storage declaration
4.Purposes of processing & user liability
5.No telemetry or advertising trackers
6.User rights and data security
7.Comtact information

1. General Provisions & Compliance

This Privacy Policy governs the processing of user information within the "GS.HTTP" mobile application (hereinafter — the "application").
This document complies with global privacy protection laws, including the EU GDPR and California CCPA, ensuring full transparency.

2. MIT License & open-sourse audit

The application’s source code is fully open-source and distributed under the MIT License.
The public status of the code allows any User or independent expert to personally inspect the software to verify the absolute absence of hidden tracking or data harvesting algorithms.

3. Data categories & local storage declaration

3.1. Personal Data: The application does not collect, request, or store any personal data (such as names, emails, phone numbers, or IP addresses) on any external servers owned by the developer.

3.2. Operational Data: To function as an HTTP client, the application processes user-inputted parameters: target URLs, HTTP methods (GET, POST, PUT, DELETE), custom Request Headers, API keys, Bearer tokens, and server response logs.

3.3. Zero-Server Architecture: All operational data and request histories are stored exclusively on you local device memory. The Developer has zero technical access to your data.

4. Purposes of processing & user liability

Technical data processing occurs solely client-side to execute network protocol commands at the User's explicit request.
The Developer does not monitor or control the network requests sent by the User.
The User assumes 100% legal liability for all network activity and compliance with target server policies.

5. No telemetry or advertising trackers

The Application is built using the Kotlin/Compose Jetpack framework and does not contain hidden third-party telemetry modules, crash reporting tools (such as Firebase, Sentry), or advertising SDKs.

6. User rights and data security

The User maintains absolute ownership over their data footprint.
You may wipe your complete request log instantly using the Application's settings or by uninstalling the software.
Data security of saved API keys depends entirely on the operating system security of the User's mobile device.

7. Comtact information

For questions regarding your privacy while using the Application, please contact the Developer at: georgsmerdov@outlook.com.
