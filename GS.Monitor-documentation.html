Вводная часть

GS.Monitor - мощный анализ HTTP-траффика/аналог android приложения GS.HTTP
GS.Monitor(далее - приложение) проверяет сайты на наличие их доступности, а также находит их по введённому значению.
Системные требования приложения разных типов установок:
Windows: 7/10/11 !> 150MB
Linux/Debian/Ubuntu: 11/12 !> 70MB

Установка приложения

Скачать приложение можно на GitHub в разделе Realeases в репозитории

Важно там будет одна версия для скачиваний , но будет много файлов, чтобы ориентироваться и не запутаться в установке, рекомендую ознакомиться с файлами каждой операционной системы:
.msi - Windows
.exe - Windows
.deb - Linux/Debian
.zip - Source code Windows/Portable version
tar.gz - Source code Linux

Запуск приложения

После того как вы установили приложение, то вы сразу же можете его запустить, никаких дополнительных настроек и компонентов для его работы не требуются.

Поиск по url

Сейчас мы рассмотрим первую работу приложения, как проверить определённый сайт на наличие его доступности и узнать дополнительные подробности.
Здесь важно правильно уловить концепцию самого приложения.
Чтобы провести первую проверку url нужнно сначала ввести его, например вводим главный сайт проекта GS -> https://gs-ht.ru.
Далее нажимаем на кнопку "Запустить скан" и видим ответ.

Введённые нами данные сохраняются в истории поиска, где мы также можем отслеживать всё что мы вводили.
История поиска хранится локально на компьютере и не передаются 3-им лицам.

Поиск по совпадениям

В приложении также доступно меню "поиск по совпадениям", главная задача которого включается найти тот или иной сайт по названию любого слова.
Здесь довольно проще, чем в панели скана url.
Здесь вам предстоит всего лишь ввести название сайта, например тот же сайт только без приписки https:// и его домена .ru.
Далее нажимаем "Найти совпадения" и видим ответ.

История поиска


Введённые нами данные сохраняются в истории поиска, где мы также можем отслеживать всё что мы вводили.
История поиска хранится локально на компьютере и не передаются 3-им лицам.

Настройки приложения


Здесь находятся настройки всего приложения в которые входят:
Сеть
Таймаут запроса - задаёт определённый промежуток времени перед запросом 5/10/15 секунд.
Авто-редирект - перенаправляет по 3xx кодам к безопасному соединению.
Безопасность
Безопасность SSl, который показывает статус ssl.
Оформление и данные
Тема оформления - меняет тему на "тёмная", "светлая", как в системе.
Очистить историю и ввод - очищает последний ввод и историю в приложении.

Частая ошибка


Здесь я хочу подметить всего лишь одну ошибку которая может возникнуть у многих пользователей при использовании приложения.
Эта ошибка связана с некоректно введённым url в меню "поиск по url", например таким https: //gs-ht.ru или таким https:/gs-ht.ru.
Казалось бы из-за одного пробела или пропускного слэша такая ошибка.
После того как вы нажмёте на ок, приложение крашнется это нормально вы его сможете снова запустить.
Ошибка происходит из-за того, что пока в приложении нет полноценной обработки ошибок, но вскоре она обязательно появится и будет просто выводить неккоректный формат ввода.
